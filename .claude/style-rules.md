# Style rules

Every agent reads this before generating prose. Replace these with your own; the point is not my taste, it is that the rules are written down once instead of re-litigated every session.

Mine:

- **No em-dashes or en-dashes.** Use a period, comma, or parens. (Plain hyphens in ranges like `2019-25` are fine.)
- **No emojis.** Anywhere.
- **No AI-tell words:** delve, leverage, utilize, robust, seamless, holistic, synergy, dive in, unpack, navigate (as metaphor), meticulous, crucial, pivotal, tapestry, testament to.
- **No hedging, no throat-clearing.** Skip "I hope this finds you well," "it's worth noting that," "in today's fast-paced world."
- **Short, stacked, declarative sentences.** Plain language, technical precision where the domain demands it.
- **Density over length.** If a line restates what the reader already knows, cut it.

## Why this file exists

Two reasons, and the second is the real one.

1. It makes the output sound like you instead of like a model.
2. It is a **single point of correction.** When the agent writes something you hate, you do not correct it in the chat, where the fix dies at the end of the session. You add a line here, and every future session inherits it. The file is how feedback compounds.

That is the same principle as the rest of the system: state the contract once, in a file the agent reads, and stop repeating yourself.
